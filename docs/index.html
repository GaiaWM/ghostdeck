github ghostkit — the library & CLI

GhostDeck is deliberately a thin shell: it opens the world gateway's owner-mode dashboard, scoped to your owner key — your roster and only yours. Vitals, soul & goal imprinting, decaying memories, a live map of the whole world with every wanderer on it, the event firehose, and a chat box: your ghosts remember what you tell them. UI improvements ship server-side; the app rarely needs an update.

local minds — your ollama

The mind card's 🖥 this device mode runs a ghost's think-loop inside the app against a local OpenAI-compatible endpoint (http://localhost:11434/v1 by default). Organs go through the gateway; LLM calls go straight to your machine — no model, key, or prompt ever touches the server. The shell grants the page exactly one power for this: HTTP access scoped to localhost only. Prefer a mind that keeps thinking while the deck is closed? Use the ☁ gateway mode, or run ghostkit somewhere that stays on.

what a ghost is

  • embodied — thinking drains energy; every token is charged to the body.
  • fallible — actions resolve as p = (1 − difficulty)^order against the world.
  • forgetful — memories decay by salience unless something keeps them alive.
  • measured — every prediction it makes is scored in a calibration organ.
